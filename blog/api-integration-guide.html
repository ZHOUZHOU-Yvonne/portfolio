第三方API对接实战：电商平台接口集成

对接第三方API是很多企业项目的核心需求——跨境ERP需要对接1688、淘宝、拼多多；SaaS产品需要对接支付宝、微信支付；数据平台需要对接各种开放平台。本文分享我帮客户对接电商API的实战经验。

1. 常见对接场景

  • 商品同步：从1688/淘宝批量拉取商品信息到自有系统
  • 订单管理：多平台订单统一管理、自动发货
  • 库存同步：实时库存更新、多仓联动
  • 物流追踪：对接快递鸟/快递100等物流API
  • 财务对账：对接支付宝/微信支付账单

2. 核心挑战与解决方案

签名认证

绝大部分中国电商API使用HMAC-SHA256或MD5签名：

import hmac, hashlib, time, urllib.parse

def sign_request(params, app_secret):
    # 1. 排序参数
    sorted_params = sorted(params.items())
    # 2. 拼接字符串
    sign_str = app_secret + ''.join(
        f'{k}{v}' for k, v in sorted_params
    ) + app_secret
    # 3. MD5签名（部分平台用HMAC-SHA256）
    return hashlib.md5(sign_str.encode()).hexdigest().upper()

# 使用示例
params = {
    'method': 'taobao.item.get',
    'app_key': '12345678',
    'timestamp': str(int(time.time() * 1000)),
    'format': 'json',
    'v': '2.0',
}
params['sign'] = sign_request(params, APP_SECRET)

限流处理

import time
from functools import wraps

def rate_limit(max_calls, period=60):
    """简单的令牌桶限流器"""
    calls = []
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            now = time.time()
            calls[:] = [c for c in calls if c > now - period]
            if len(calls) >= max_calls:
                sleep_time = calls[0] + period - now
                if sleep_time > 0:
                    time.sleep(sleep_time)
            calls.append(time.time())
            return func(*args, **kwargs)
        return wrapper
    return decorator

@rate_limit(max_calls=30)  # 每分钟最多30次

数据同步策略

  1. 增量同步：使用updated_at字段，只拉取变更数据
  2. 分页处理：page_size+page_no，处理大数据量
  3. 失败重试：指数退避，最多3次
  4. 幂等设计：使用唯一业务键，防止重复写入

4. 对接前的准备清单

  1. 确认API文档是否齐全（有没有中文版）
  2. 申请AppKey/AppSecret（通常需要企业资质）
  3. 准备测试环境（沙箱/sandbox）
  4. 确认调用频率限制（QPS/day limit）
  5. 确认数据字段映射（自己的系统和API返回的数据怎么对应）
