微信小程序开发全流程：从注册到上线

2026-06-08 · 10 min read

微信小程序月活超10亿，是企业触达用户的黄金渠道。本文分享从零开发一个小程序的完整流程，适合有前端基础的开发者。

1. 注册与准备

  1. 访问 mp.weixin.qq.com 注册小程序账号（需邮箱+手机号）
  2. 下载微信开发者工具
  3. 在后台获取 AppID
  4. 配置服务器域名白名单（request、uploadFile等合法域名）

2. 项目结构

miniprogram/
├── app.js          # 小程序入口
├── app.json        # 全局配置
├── app.wxss        # 全局样式
├── pages/          # 页面目录
│   ├── index/      # 首页
│   │   ├── index.wxml   # 页面模板
│   │   ├── index.js     # 页面逻辑
│   │   └── index.wxss   # 页面样式
│   └── detail/     # 详情页
├── components/     # 自定义组件
├── utils/          # 工具函数
└── images/         # 图片资源

3. 核心开发

页面模板 (WXML) — 类似HTML，使用小程序专用组件：

<view class="container">
  <swiper indicator-dots autoplay>
    <swiper-item wx:for="{{banners}}" wx:key="id">
      <image src="{{item.image}}" mode="aspectFill"/>
    </swiper-item>
  </swiper>
  <view class="service-list">
    <view wx:for="{{services}}" wx:key="id" bindtap="onTap">
      {{item.name}} - ¥{{item.price}}
    </view>
  </view>
</view>

4. 常用功能实现

  • 用户登录：wx.login()获取code → 服务端换取openid
  • 支付对接：wx.requestPayment() → 服务端调用微信支付API
  • 地图定位：wx.getLocation() + map组件
  • 图片上传：wx.chooseImage() + wx.uploadFile()
  • 分享裂变：onShareAppMessage() 自定义分享卡片

5. 审核与上线

提交审核前检查清单：

  1. 所有功能可正常使用（无空白页、无报错）
  2. 类目选择正确（企业/个体户对应不同类目）
  3. 隐私协议完整（设置→基本设置→隐私保护指引）
  4. 无测试数据残留
  5. 服务器域名已配置且HTTPS可用

审核通常1-3个工作日。首次提交建议选择"不发布"，审核通过后手动发布。

6. 小程序 vs H5 vs App

给潜在客户的技术选型建议：

  • 预算有限、需要微信生态 → 小程序
  • 需要SEO、分享到朋友圈 → H5网页
  • 复杂交互、推送通知 → App
